## 테스트 프레임워크
- **pytest** (Python 3.11+)
- AAA 패턴, Given/When/Then은 **주석만**
- naming: test_<test_id>_<scenario> (예: test_u_in_04_zero_empty_cells_returns_e002)

## RED Skeleton 규칙
1. 테스트 파일·클래스·함수·fixture **구조만** 작성
2. 각 테스트 본문은 **의도적 실패 한 줄만**:
   `pytest.fail("RED: <Test ID> — <한 줄 설계 요약>")`
3. production import는 **허용** (Report/08과 동일):
   - Boundary: `from boundary.input_validator import InputValidator` 등
   - Entity: `from entity.services... import ...` (아직 없으면 collection ERROR도 RED로 인정)
4. Arrange/Act 코드(validator 생성, grid fixture 사용)는 **주석으로만** 표시 가능
5. Then 검증은 **작성하지 않음** — pytest.fail()만
