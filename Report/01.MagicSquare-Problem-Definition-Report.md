## Executive Summary

본 과제의 표면 목표는 “4×4 마방진을 완성하는 프로그램”이지만, 분석 결과 **진짜 문제**는 다음과 같이 정의된다.

> 고정된 4×4 정수 격자에 대해, 규칙 준수 여부를 **일관되게 판정**하고, 필요 시 **유효한 배치를 반복적으로 산출**하며, 판정·산출이 **동일 기준으로 재현·검사** 가능해야 한다.

핵심은 격자 한 장의 완성이 아니라, **제약·불변 조건·계약(입출력 의미)·검증 우선** 사고를 훈련하는 것이다.

---

## STEP 1 — Observation (관찰)

### 1.1 현재 해결하려는 상황

4×4 격자에 1~16을 한 번씩 배치했을 때, 행·열·대각선의 합이 모두 같아지는 배치를 **찾거나 검증**하려는 상황이다.

| 관찰 | 내용 |
|------|------|
| 입력 공간 | 16칸에 1~16 순열 — 경우의 수가 매우 큼 |
| 제약 | 행·열·대각선 합 동일이 동시에 적용 |
| 불확실성 | 임의 배치는 대부분 조건 불만족 |
| 의도 | 유효 배치 존재·생성·주어진 배치 검증 |

→ 단순 계산이 아닌 **규칙을 만족하는 구조의 탐색·확인** 문제.

### 1.2 4×4를 다루는 관찰 가능한 동기

- **규모:** 3×3보다 제약 많고, 5×5보다 다루기 쉬움  
- **패턴:** 짝수 차수·대칭·블록 등 여러 기법 교차  
- **검증 연습:** 만들기 vs 검증하기 분리에 적합  
- **과제 맥락:** 2차원 구조·조건·탐색을 한 과제에 묶기 쉬움  

### 1.3 학습·시스템 설계 맥락

- **학습:** 이산수학·퍼즐, 알고리즘 입문, 요구사항→명세→검증  
- **시스템:** CSP 축소판, 생성기/검증기 분리, n×n 확장 전 프로토타입  

**한 줄:** 16칸에 1~16을 놓고 여러 방향의 합이 동시에 맞는지 다루는 **작은 규모의 제약·조합 문제**.

## STEP 2 — Why #1 (왜 완성해야 하는가)

### 2.1 가정한 답 (A)

> 4×4에 1~16을 한 번씩 넣었을 때, 모든 행·열·대각선의 합이 같은 **올바른 배치**를 얻어야 검증·제출·시연이 가능하다.

완성은 목적이 아니라 **규칙을 만족하는 산출물 확보**의 수단.

### 2.2 드러난 불편함

1. **완성 vs 검증 가능** 혼동 — 16칸 채움 ≠ 제약 만족  
2. **과정·근거 소실** — 최종 격자만 남고 설명·수정 어려움  
3. **해의 다양성** — 유효 배치는 여러 개, “하나의 정답” 오해  
4. **비효율적 탐색** — “무조건 채움”은 문제 규모 대비 접근이 비대해짐  
5. **빈약한 실패 피드백** — 어느 제약이 깨졌는지 모름  

### 2.3 구조적 문제

| 문제 | 설명 |
|------|------|
| 목표가 단일 산출물로 고정 | 생성·검증·설명·확장이 한 덩어리 |
| 완료 조건 암묵적 | “다 채움”과 “합 일치” 미분리 |
| 정답 다양성 미정의 | 아무 유효 배치 vs 특정 패턴 불명 |
| Why가 How 선점 | 검증 우선·구성 절차 분기 가려짐 |

## STEP 3 — Why #2 (왜 프로그램인가)

| 관점 | 단순 계산 | 프로그램 |
|------|-----------|------------|
| **반복 가능성** | 한 번의 손해, 이식 어려움 | 동일 절차·확장·재실행 |
| **검증 자동화** | 최종 합 위주, 부분 검사 약함 | 행·열·대각·중복 일관 감사 |
| **오류 방지** | 배치·합·중복 실수 취약 | 실수 유형을 명시적으로 차단 |
| **규칙 기반 사고** | 결과·패턴 중심 | 제약·절차·검증 분리·명문화 |

**한 줄:** 단순 계산은 “한 개의 올바른 격자”, 프로그램은 “같은 규칙으로 여러 번, 검증 가능하게, 규칙을 고정”.

### 구조적 긴장

- 프로그램 ≠ 자동으로 올바른 생성  
- 규칙 훈련 vs 과제 완료(표 복사) 충돌 가능  
- 4×4 한 장만이면 손검증이 더 빠를 수 있음 → 정당성은 **반복·검증·확장·명세화**

## STEP 4 — Why #3 (왜 TDD인가)

### 4.1 통제되어야 하는 것

- 제약 정의(대각선 범위, 부분 마방진 여부)  
- 검증 vs 생성 역할  
- 유효성 판정 기준(채움 ≠ 유효)  
- 숫자 집합(1~16, 중복, 빈 칸)  
- 합의 기준값(4×4 정상 시 34)  
- 변경 시 깨지면 안 되는 **계약**  

→ 통제 대상은 “예쁜 마방진 기술”이 아니라 **“무엇을 올바르다고 부를지”**.

### 4.2 불변 조건 (요약)

**도메인:** 집합 {1…16}, 모든 검사 선의 합 동일(34), 동일 해(회전·대칭) 취급은 요구로 고정  

**시스템:** 판정 일관·충실, 산출-판정 정합, (선택) 실패 사유  

**TDD 연결:** 테스트 = 불변 목록; 반례·유효 예시·경계 격자로 고정  

### 4.3 입출력이 명확해야 하는 이유

- 테스트 = 관찰 하나; 모호한 입출력은 RED-GREEN-REFACTOR 불가  
- 검증기 입출력을 먼저 고정하는 것이 자연스러움  

**핵심 발견:** TDD는 “맞는 격자”보다 **깨지면 안 되는 제약**과 **모듈 간 계약**을 먼저 통제하기 위함.

## STEP 5 — 진짜 문제 정의

### 5.1 표면 문제 정의 (잘못된 정의)

> 4×4 격자에 1~16을 채워, 모든 행·열·대각선의 합이 같은 마방진을 **완성하는 프로그램**을 만든다.

**위험:** 채움=성공 오해, 역할 혼합, 프로그램 정당성·품질 기준 부재.

### 5.2 개선된 문제 정의 (정확한 정의)

> **고정된 크기(4×4)의 정수 격자에 대해**, 주어진 배치가 “1~16을 각각 한 번만 사용하고, 정의된 모든 선(행·열·대각선)의 합이 동일한가”를 **일관되게 판정**할 수 있어야 하며, 필요 시 그 규칙을 만족하는 **유효 배치를 반복적으로 산출**할 수 있어야 한다. 판정·산출은 **동일 기준으로 재현·검사** 가능해야 한다.

**미결정(요구사항 단계에서 확정):** 유효 배치 개수, 회전·대칭 동일 해, 미완성 격자 처리.

### 5.3 핵심 Invariant

| # | 불변 |
|---|------|
| A1 | 완전 배치 시에만 “완전한 배치” 판정(범위 한정 시) |
| A2 | 값 집합 = 1~16 각 1회 |
| A3 | 검사 대상 선의 합 모두 동일 |
| A4 | 공통 합 = 34 (정상 4×4 정의 시) |
| B5 | 동일 격자·규칙 → 동일 판정 |
| B6 | 유효 판정 → A2·A3(·A4) 충족 |
| B7 | 산출물은 판정 통과 |
| B8 | 무효 시 무효임이 거짓 아님 |
| C9 | “유효” 의미가 구현보다 먼저, 예시·반례로 표현 |
| C10 | 판정 vs 산출 논리적 분리 |

### 5.4 훈련하려는 사고 능력

- 제약 명문화  
- 성공·실패 분리 (채움 ≠ 규칙 통과)  
- 불변 조건 우선 사고  
- 계약(입·출력 의미) 고정  
- 검증 우선  
- 반례·경계 사고  
- 반복 가능한 신뢰  
- 관심사 분리  

**훈련 목표 한 줄:** 답 한 장이 아니라, **옳은 상태**를 불변·계약으로 고정하고 판정·산출을 신뢰 가능하게 만드는 사고.

## 표면 vs 개선 — 최종 대조

| 구분 | 표면 | 개선 |
|------|------|------|
| 초점 | 격자 완성 | 규칙 준수 판정 + (선택) 산출 |
| 성공 | 16칸 채움 | 불변 조건 충족 |
| 도구 | 한 번의 답 | 반복·검증·명세화 |
| 학습 | 패턴 암기 | 제약·불변·계약·검증 우선 |
