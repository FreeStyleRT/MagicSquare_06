# MagicSquare Cursor Rules 분리 작업 보고서

작성일: 2026-05-28  
저장소: `MagicSquare_16`  
문서 목적: `.cursorrules` 기반 규칙을 `.cursor/rules/*.mdc` 구조로 분리/적용한 결과를 기록한다.

---

## 1. 작업 배경

기존 단일 규칙 파일(`.cursorrules`)을 주제별 Rule 파일로 분리해
가독성, 유지보수성, 규칙 선택 적용성을 높이고자 했다.

---

## 2. 수행 범위

이번 작업에서 수행한 범위:

1. 기존 `.cursorrules` 분석
2. 분리 대상 규칙 5종 설계
3. `.cursor/rules` 디렉터리 생성
4. `.mdc` 규칙 파일 5개 생성
5. 파일 생성 결과 확인

---

## 3. 생성 결과

생성 파일:

- `.cursor/rules/magicsquare-project.mdc`
- `.cursor/rules/magicsquare-ecb-architecture.mdc`
- `.cursor/rules/magicsquare-forbidden.mdc`
- `.cursor/rules/magicsquare-python-code-style.mdc`
- `.cursor/rules/magicsquare-tdd-testing.mdc`

규칙 분리 기준:

- `project/file_structure/ai_behavior 일부` -> `magicsquare-project.mdc`
- `ECB 레이어/의존 방향` -> `magicsquare-ecb-architecture.mdc`
- `금지 패턴` -> `magicsquare-forbidden.mdc`
- `Python 스타일` -> `magicsquare-python-code-style.mdc`
- `TDD 단계/pytest 규칙` -> `magicsquare-tdd-testing.mdc`

---

## 4. Rule 메타 설정

- 공통 전역 규칙:
  - `magicsquare-project.mdc`: `alwaysApply: true`
  - `magicsquare-ecb-architecture.mdc`: `alwaysApply: true`
  - `magicsquare-forbidden.mdc`: `alwaysApply: true`
- 파일 매칭 규칙:
  - `magicsquare-python-code-style.mdc`: `globs: "**/*.py"`, `alwaysApply: false`
  - `magicsquare-tdd-testing.mdc`: `globs: "{tests,magic_square}/**/*.py"`, `alwaysApply: false`

---

## 5. 품질 점검

확인 항목:

- `.cursor/rules` 경로 및 파일 생성 여부 확인 완료
- 생성된 5개 파일명/위치 확인 완료
- 변경 경로 기준 린터 오류 없음

---

## 6. 비고

- 사용자 요청에 따라 규칙 파일의 구조/내용은 확정안 그대로 반영했다.
- 후속으로 필요 시 파일별 줄 수(30~50) 미세 조정 및 문장 압축이 가능하다.
