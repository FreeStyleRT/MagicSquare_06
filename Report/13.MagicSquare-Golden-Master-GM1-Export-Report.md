## 1. GM-1 Golden Master

### 시나리오 (5)

| Section ID | 입력 요약 | committed baseline |
|---|---|---|
| `normal_success` | D-01 (0,2,3,13 / … / 15,0) | `Output: [1,1,16,4,4,1]` |
| `reverse_success` | D-02 (16,2,3,13 / … / 15,0) | `Output: [3,3,6,4,4,1]` |
| `invalid_blank_count` | 빈칸 3개 | `Error: INVALID_BLANK_COUNT` |
| `duplicate_number` | 비0 중복 2 | `Error: DUPLICATE_NON_ZERO` |
| `no_valid_solution` | G3 placeholder | `Error: NO_VALID_SOLUTION` |

### 캡처 방식

- **Control 진입점:** `ResolveUseCase.execute(grid)`
- **성공:** `list[int]` → `Output:\n[r,c,n,r,c,n]` (1-index, compact)
- **실패:** `FailureResponse.code` → `Error:\n{code}`
- stdout 미사용 (ECB: Boundary GUI/CLI 독립)

### Approve 패턴

| 조건 | 동작 |
|---|---|
| 기준 파일 없음 | 현재 출력으로 자동 생성 |
| `--golden-approve` / `--approve` | baseline 덮어쓰기 |
| 기준 파일 있음 (일반) | 전체 문서 문자 비교 |
| 불일치 | `difflib.unified_diff` 후 FAIL |

### 운영 명령

```powershell
python scripts/generate_golden_master.py --approve
python scripts/generate_golden_master.py
python -m pytest tests/boundary/test_golden_master.py
python -m pytest tests/boundary/test_golden_master.py --golden-approve
```
